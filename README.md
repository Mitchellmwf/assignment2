#Marie-Eve Bouchard Reflection
What the team learned:
What we learned was how to bring together everything we learned in class to create a simple database management system. We also learned the process of working together as a team to complete the project.

How members collaborated:
There were three members in our group, and we collaborated by dividing tasks and helping each other when needed.

Challenges faced:
One challenge we faced was that I was working online, which made it harder to show certain things to my teammates. For example, explaining how to work in different branches in GitHub and how to merge them was difficult since the rest of my teammates had never done that before. Because of this, it would have been challenging to teach it remotely. Instead, we decided to work on commits one by one so the commit history would still be clear. Some of the steps were also confusing at first, but we eventually figured them out.

How they solved problems:
We solved problems by thinking of alternative ways to do things, such as changing how we handled the commits. When I got stuck, I also reviewed code from previous classes and assignments to check if I had missed something. In some cases, I had missed some things that took longer especilly plymorphism and about abstract classes and make it work all togther, so I made sure to carefully review everything.

What could be improved next time:
Next time, the process could be improved by having clearer steps from the beginning of the project. It would also be helpful to have some suggested resources or links that we could look at for additional information that relates to what we are working on our should know such as how to branches in github. This could make it easier to understand certain parts of the project and help the group work more efficiently.
